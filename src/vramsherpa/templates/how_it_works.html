How VRAM Sherpa estimates fit (Policy v0.1)

VRAM Sherpa estimates whether a model variant can fit on your available VRAM for local inference. It is a planning estimate, not a benchmark.

Inputs

  • available_vram_gb from GPU selection or manual VRAM input
  • context_tokens from presets (2048 default, 4096, 8192)
  • Variant and model metadata: params_b, bits_effective, kv_gb_per_1k_ctx

Formula

weights_gb = params_b * bits_effective / 8 * 1.10

kv_cache_gb = kv_gb_per_1k_ctx * (context_tokens / 1000)

runtime_overhead_gb = 1.0 + 0.05 * params_b

required_vram_gb = weights_gb + kv_cache_gb + runtime_overhead_gb

Reserve buffer

reserve_gb = max(0.75, 0.10 * available_vram_gb)

Classification

  • Fits: required_vram_gb <= available_vram_gb - reserve_gb
  • Tight: available_vram_gb - reserve_gb < required_vram_gb <= available_vram_gb
  • Won't fit: required_vram_gb > available_vram_gb

Limitations

Real outcomes vary by runtime, driver stack, memory fragmentation, and CPU/GPU offload behavior. This tool is not a benchmark and does not guarantee performance.
